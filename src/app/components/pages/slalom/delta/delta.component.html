Delta - Itegrated Crew Hub (ICH)

July 2019 - March 2020
At Delta, I worked in the Emerging Technologies (EMT) program, primarily on the ICH Project. ICH is an internal tool for Delta used by crew schedulers. It is used to help visualize crew member' schedules and identify problems. The system can find available resources to replace crew (ie if a pilot is delayed and cannot pilot a connecting flight). The tool can generate different scenarios (ie deicing taking a longer amount of time) to find potential risks for the crew.

D3 Visualizations. The main components of the project was a variety of dashboards displaying different metrics on pilots and flight attendants statuses. Many of these visualizations were made using D3 (Data-Driven Documents) and were interactive to drill into specific sections at a simple click. These visuals are shown in the pictures, such as the multi bar graph, gantt charts, pie charts, and heat maps.
What If - Gantt Chart
The unique aspect about the D3 visualizations is that the code was written in such a way that they are reusable and you can specify which components you would want to display. For example, the blue vertical line in a graph indicates the current time and shades everything in the past; this could be toggled on/off and even customized based on different options provided to the component.

Angular Frontend. The Frontend of this application was made using Angular. We used Angular's reactive forums to build a filtering system that saved a user's preferences and provided a link to share the preferences with other co-workers. This application was part of the EMT program, which originally used a mono-repository for all sub-projects. We improved the structure by breaking out each sub-project into it's own repository and having the main code-base load in different sub-projects whenever needed.
Filter Forum Modal
Java Backend. The Backend for the project was 2 Java Spring boot applications. The first was a BFF (Backend to Frontend) Application that provided extra security and results caching. The second was a normal backend server that recieved request and interacted with a Teradata database to retrive information for the user.
